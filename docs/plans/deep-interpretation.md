# 深度断卦规划

**日期**: 2026-06-11
**路线**: 增强规则 + AI 权衡（规则算准结构化要素，AI 综合权衡表达）
**定位**: 比当前启发式规则更专业，但不追求"专业断卦引擎"的全流派完整性

---

## 一、职责边界（核心设计）

三层各司其职，**不越界**：

```
core（@najia/core）          → 算「客观事实」：纯排盘 + 可确定计算的关系
  └─ rule-engine（server）   → 算「断卦要素」：用神、旺衰、空破、动变关系，结构化输出
       └─ AI（LongCat）       → 做「综合权衡 + 表达」：基于结构化要素，权衡矛盾信号，人话输出
```

判定原则：
- **能确定算的，规则算**（旺衰、生克、空破、动变方向）——不交给 AI，避免 AI 算错基础事实
- **需权衡的，AI 判**（多个矛盾信号谁主导、应期、最终吉凶倾向）——规则只提供候选，不硬下结论
- **会冲突的规则，不在规则层强行打分**——把冲突信号都列出来交给 AI 权衡

## 二、当前缺口（要补的断卦要素）

core 已产出的原料（够用）：
- 六亲 qin6 / 纳甲 qinx / 六神 god6 / 世应 shiy / 动爻 dong
- 月令旺衰 yue_ling / 月破 yue_po / 旬空 xun_kong / 月建 yue_zhi / 日辰 ri_chen
- 变卦 bian / 伏神 hide

**rule-engine 目前只做了**：五行分布、世应距离生克、月令聚合、动爻数量、粗打分吉凶。

**缺的关键断卦要素**（本次要补）：

| 要素 | 说明 | 依赖原料 | 难度 |
|------|------|---------|------|
| **用神取用** | 按问事类别定用神六亲 | qin6 + 问事类别 | 中（需类别输入） |
| **用神定位** | 找出用神在哪几爻（可能多现/不上卦） | qin6 | 低 |
| **用神旺衰** | 用神爻的月令/旬空/月破综合状态 | yue_ling+yue_po+xun_kong | 低 |
| **动爻作用关系** | 动爻对用神的生/克/合/冲，回头生克 | dong+qinx+bian | 高 |
| **进退神** | 动爻化进/化退（同五行地支前进后退） | dong+qinx+bian.qinx | 中 |
| **伏神/飞神** | 用神不上卦时看伏神能否出现 | hide | 中 |
| **应期候选** | 提供逢值/逢冲/出空等候选时间点（不下定论） | 用神地支+空破 | 高 |

## 三、用神取用：怎么知道"问什么"

这是路线成败的关键。三种方案，**规划建议方案 A**：

### 方案 A：前端加"问事类别"下拉（推荐）
- 排盘界面加一个选择器：事业/财运/婚恋/健康/学业/出行/官非/失物/综合
- 类别 → 用神映射表（规则确定）：

  | 类别 | 主用神 | 备注 |
  |------|--------|------|
  | 财运/求财 | 妻财 | 兼看官鬼（财之源动力） |
  | 事业/官运 | 官鬼 | 兼看父母（文书/印绶） |
  | 婚恋（男问） | 妻财 | |
  | 婚恋（女问） | 官鬼 | |
  | 健康/疾病 | 世爻 + 子孙（医药） | 忌官鬼（病符） |
  | 学业/考试 | 父母（文书） + 官鬼（功名） | |
  | 出行 | 世爻 | 忌父母（阻滞） |
  | 失物 | 妻财（财物） | |
  | 综合/其他 | 世爻 | 取不出明确用神时兜底 |

- 优点：用神取用 100% 确定，规则层可靠
- 成本：前端加一个字段 + 契约加 category 字段，工作量小

### 方案 B：AI 从 question 文本判类别
- 让 AI 先分类，再取用神
- 缺点：多一次 AI 往返 / 或在同一 prompt 里做，分类错则全错；规则层失去确定性

### 方案 C：两者结合
- 前端可选类别（默认"综合"），未选时由 AI 兜底判断
- 折中，但增加复杂度

**建议 A**：最符合"规则算准"的路线定位，成本最低。

## 四、实施分期

### 阶段 1：用神取用 + 定位 + 旺衰（地基，可独立交付）
1. core 或 server 加「问事类别 → 用神」映射表
2. 前端排盘界面加类别选择器，契约加 `category` 字段
3. rule-engine 新增：
   - `findYongShen(qin6, category)` → 用神六亲 + 所在爻位（含多现/不上卦判断）
   - `analyzeYongShenStrength(用神爻, yue_ling, yue_po, xun_kong)` → 用神旺衰状态（旺相休囚死 + 空/破标记）
4. 验证：单元测试覆盖各类别取用 + 用神旺衰边界（空、破、旺、衰）

### 阶段 2：动变关系（断卦核心，难度最高）
5. rule-engine 新增：
   - 动爻对用神的作用（生/克/合/冲）
   - 回头生克（变爻对动爻）
   - 进退神判断
6. 验证：构造已知卦例（教材标准案例）对照验证

### 阶段 3：伏神 + 应期候选
7. 用神不上卦 → 伏神能否透出（飞神生克伏神）
8. 应期候选点生成（逢值/逢冲/出空，列候选不下定论）

### 阶段 4：AI 权衡层升级
9. 把结构化断卦要素喂给 AI（替代当前的原始字段罗列）
10. prompt 升级：要求 AI 基于「用神旺衰 + 动变 + 矛盾信号」做权衡，明确哪个因素主导
11. 用 interpret-samples 测试集对比升级前后效果

## 五、风险与边界

- **流派差异**：六爻有多个流派（京房/野鹤/增删卜易…），规则取主流（野鹤派为主），文档标注依据，不追求各派全覆盖
- **应期最不可靠**：只提供候选时间点，明确标注"参考"，不下确定结论
- **规则算错比 AI 说错更严重**：规则层每个新要素必须有单元测试 + 已知卦例对照，对拍思路延续
- **不做的**：六神断应（青龙主喜庆等)的细化、卦身、星煞、纳音——这些流派分歧大、收益低，暂不纳入
